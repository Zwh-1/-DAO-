# 成员权限（member）

**完整可调用接口列表**：凡在 [后端路由矩阵.md](./后端路由矩阵.md) 中标记为仅 **`requireAuth`**（且无 `+oracle` / `+arbitrator`）的端点，均属于「已登录成员默认可用」范围；与下列说明一致。新增或变更后端路由时，请同步更新该矩阵。

下面按本仓库实际代码说明「成员（member）」是什么，以及能做什么 / 不能做什么。

## 1. 「成员」是怎么来的？
首次登录时，POST /v1/auth/verify 会 getOrCreateMemberProfile(address)；在内存里新建用户时默认带上 roles: ["member"]（见 storage.js 里 getOrCreateMemberProfile / getMemberRoles）。
文档 链上角色与权限.md 里也写了：在 ROLES_SOURCE=chain 时，member 是基线，任意通过 SIWE 的地址都会给（与其它链上角色叠加）。
所以：登录钱包并完成 SIWE 后，至少会有 member，这是「普通已登录用户」的基线身份，不是额外审批出来的角色。

## 2. 后端有没有单独写「member 权限」？
没有。 中间件里只有：

requireAuth：只要有有效的 JWT（登录即通过）。
requireRole("oracle")、requireRole("arbitrator") 等：必须 JWT 的 roles 里包含对应角色。
代码里 没有出现 requireRole("member")。
因此：「member 的权限」在实现上等价于「已登录用户能访问的所有 requireAuth、且不要求其它角色的接口」。
member 更多是标签 + 前端切换身份时的默认档，而不是单独一套后端校验逻辑。

## 3. 仅持有 member（或只有基线角色）时，大致能用什么能力？
在 只登录、没有 oracle / arbitrator / … 的前提下，只要接口 只写了 requireAuth、没有 requireRole(...)，你都可以用。例如（不完全列举）：

方向	说明
申领	POST /v1/claim/propose 等（需登录 + 业务校验，如地址与 JWT 一致）
成员	POST /v1/member/wallets/bind（绑定钱包）等需登录的接口
治理	POST /v1/governance/propose、POST /v1/governance/vote（仅要求登录）
通道	/v1/channel/* 中带 requireAuth 的接口
多签	/v1/multisig 下需登录的接口
声誉	如 POST /v1/reputation/verify、GET /v1/reputation/:address 等仅 requireAuth 的
其它	部分 security、ai 里只要求登录的路由
不能用的，是明确写了 requireRole("oracle")、requireRole("arbitrator") 等的接口，例如：

身份/SBT 管理、多数 Oracle 上报/签名（identity.routes.js、oracle.routes.js 里带 Oracle 的）。
仲裁提交/揭示（member.routes.js 里 arb/commit、arb/reveal 需要 arbitrator）。
声誉里「锚定历史行为」等需要 oracle 的接口。

## 4. 前端导航里「成员」能看到什么？
NavItems.ts 里：没有写 requiredRole 的项（门户、钱包、理赔申请、成员中心子菜单）对任意有角色列表的用户（包括只有 member）都会按角色过滤规则显示；
仲裁 / 挑战者 / 预言机 / 守护者 等带 requiredRole 的入口，需要 对应角色 才会出现。

说明：主导航 DAO 分组使用 `UserRole.DAO`（`dao`），与后端 JWT `RoleId` 一致；若侧栏仍不显示 DAO 入口，请确认 JWT `roles` 中是否包含 `dao`。详见 [身份切换与 activeRole.md](./身份切换与%20activeRole.md)。

## 5. 一句话总结

member：自动给的普通用户基线，表示「已通过 SIWE 的账号」。
后端权限：不按 member 单独鉴权；登录 = JWT 才能访问需登录接口；更高操作靠 oracle / arbitrator / … 限定。
若你希望「成员只能读、不能投票」这类 细粒度策略，需要 新增规则（例如对 governance/vote 再加 requireRole 或业务策略），当前代码 没有 把「member」单独限制成只读。
