# 后端路由矩阵（HTTP 鉴权）

本文档为 **TrustAid 后端 API 鉴权类型的单一事实来源**：与实现一致，修改路由时请同步更新本表。

**鉴权类型说明**

| 类型 | 含义 |
|------|------|
| `公开` | 无需 JWT、无需 Admin Token |
| `requireAuth` | 请求头 `Authorization: Bearer <JWT>`，JWT 内 `roles` 通常含 `member` 等 |
| `requireAuth+oracle` | 在 `requireAuth` 基础上，`requireRole("oracle")` |
| `requireAuth+arbitrator` | 在 `requireAuth` 基础上，`requireRole("arbitrator")` |
| `requireAdmin` | 管理员 Token（见 [`backend/src/middleware/auth.js`](../../backend/src/middleware/auth.js) `requireAdmin`），**与 JWT 角色无关** |

**重要**：`/v1/guardian/*` 前缀路由均使用 `requireAdmin`，**不是** `requireRole("guardian")`。链上/产品中的「守护者」JWT 与「守护者模块 HTTP 接口」当前为两套机制。

**维护约定**：新增或变更 [`backend/src/routes/*.routes.js`](../../backend/src/routes/) 或 [`backend/src/server.js`](../../backend/src/server.js) 中的路由时，请更新本文件对应行。

## JWT 角色与路由层（更新说明）

| 角色 | 说明 |
|------|------|
| `member` | 无单独 `requireRole("member")`；多数「仅登录」路由依赖 `requireAuth`（见 [成员权限.md](./成员权限.md)） |
| `challenger` | `/v1/challenge/*` 使用 `requireRole("challenger")`；`init` 须 `challenger` 与 JWT 地址一致 |
| `dao` | `/v1/governance/propose`、`/vote` 及 `/v1/multisig/governance/*` 写操作已绑定 `requireRole("dao")` |
| `guardian`（JWT） | `/v1/guardian/*` HTTP 仍使用 **`requireAdmin`**，与 JWT `guardian` 分离；见 [权限模型.md](../安全/权限模型.md) |

**架构决策全文**：[权限模型.md](../安全/权限模型.md)

**前端对齐**：主导航 DAO 页与后端 `dao` 角色一致；见 [`NavItems.ts`](../../frontend/features/governance/NavItems.ts)。
