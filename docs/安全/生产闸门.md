# 生产环境闸门与安全口径

本文对应后端实现中的硬约束（与自检清单对齐）。

## 认证与管理员

| 变量 | 含义 |
|------|------|
| `BYPASS_AUTH` | 禁止在生产启用（见 `validateConfig`） |
| `ALLOW_BYPASS_BODY_AUTH` | `BYPASS_AUTH=1` 时，若为 `1` 才允许无 Bearer、用 body 伪造地址；默认应关闭。若你曾依赖「无 JWT 仅 body」联调，请显式置 `1` 或改为始终带 `Authorization: Bearer` |
| `ALLOW_INSECURE_ADMIN` | 非 production 且无 `ADMIN_TOKEN` 时，若为 `1` 才放行 `requireAdmin`；默认关闭（返回 503） |
| `ADMIN_TOKEN` | 生产必填 |

## 链上与「假成功」

- **匿名申领** `POST /v1/anonymous-claim/claim`：production 且无 RPC/relayer/`ANONYMOUS_CLAIM_ADDRESS` → **503**，禁止纯链下成功语义。
- **多签治理** `queue` / `execute` / `cancel`：production 且无 Governance 中继 → **503**（`5043`）。

## Nullifier 服务端派生

- `POST /v1/security/nullifier/derive`：production 默认 **403**，除非 `ALLOW_SERVER_NULLIFIER_DERIVE=1`（不推荐）。

## 部署（合约）

- `contracts/scripts/deploy.mjs`：在常见主网/L2 `chainId` 上禁止默认部署 **MockGroth16Verifier**，除非 `FORCE_MOCK_VERIFIER=1`。本地链（如 31337）不受影响。

## ZK 验证语义

- 后端 `zkVerify.service.js` 仅为形状预校验；**链上 Verifier 为真理源**。可选链下 `groth16.verify` 需独立设计与 vk 版本管理。
