# 去中心化社区互助平台

## 各角色子系统完整设计文档（毕业设计版）

---

## 一、设计概述

本系统基于区块链与 DAO 治理思想，采用“角色子系统化（Domain Module）”架构，将传统的单一页面角色设计升级为**多模块、强解耦的业务子系统结构**。

每个角色不再仅对应一个功能入口，而是形成**完整业务闭环系统**，具备独立页面、数据结构、状态机与权限控制。

---

```
 ├─ 预言机合约

支撑层
 ├─ IPFS（存储）
 ├─ The Graph（索引）
 ├─ Chainlink（VRF + Oracle）
```

## 三、仲裁员子系统（Arbitration System）

### 3.1 子系统目标

实现公平、匿名、去中心化裁决机制。

---

### 3.2 页面结构

```
仲裁中心
 ├─ 概览 Dashboard
 ├─ 待裁决案件
 ├─ 案件详情
 ├─ 证据查看
 ├─ Commit 投票
 ├─ Reveal 投票
 ├─ 仲裁记录
 ├─ 声誉系统
 ├─ 奖励中心
```

---

### 3.3 核心功能

#### 1. 案件管理

* 展示被随机选中的案件
* 显示裁决阶段与时间限制

#### 2. 裁决流程（核心机制）

```
进入仲裁
 → Commit（提交加密投票）
 → Reveal（公开投票）
 → 共识计算
 → 执行结果
```

#### 3. 证据访问控制

* 多签授权解密
* 限时访问
* 访问日志记录

#### 4. 声誉系统

* 投票一致率
* 恶意投票记录
* 信用评分变化

---

### 3.4 数据结构

```ts
Case {
  id: string
  claimant: address
  amount: number
  status: string
  commitDeadline: number
  revealDeadline: number
}
```

---

### 3.5 合约接口

```solidity
commitVote(caseId, hash)
revealVote(caseId, choice, salt)
claimReward()
```

## 十三、设计优势总结

本设计具有以下优势：

1. **模块化清晰**：每个角色独立子系统
2. **高扩展性**：新增角色无需影响其他模块
3. **安全性强**：权限隔离 + 合约控制
4. **符合区块链思想**：去中心化治理 + 透明可追溯
5. **适用于毕业设计**：具备完整系统深度

---

## 十四、结论

通过将各角色升级为独立子系统，本平台从传统单页面应用升级为**多域协同的去中心化系统架构**，显著提升了系统的工程价值与研究深度，为区块链互助平台提供了一种可落地的设计范式。
