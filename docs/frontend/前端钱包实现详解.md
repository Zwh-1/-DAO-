## 一、钱包架构总览

### 1.1 技术栈

TrustAid 前端钱包采用以下技术栈：

| 层级 | 技术 | 说明 |
|------|------|------|
| **状态管理** | Zustand | 轻量级状态管理库 |
| **钱包 SDK** | @trustaid/wallet-sdk | 自研钱包 SDK |
| **区块链** | Ethers.js | 以太坊交互库 |
| **UI 框架** | React 18 + Next.js 14 | 服务端渲染 |
| **样式方案** | Tailwind CSS | 原子化 CSS |
| **签名协议** | SIWE (Sign-In with Ethereum) | 以太坊登录 |

### 1.2 架构图

```
┌──────────────────────────────────────────────────┐
│              应用层 (Pages)                       │
│  HomePage, ClaimPage, MemberPage, DAOPage       │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│           组件层 (Components)                     │
│  Topbar, WalletButton, WalletStatusBadge        │
│  ConnectWalletModal（MetaMask / WC / 内置钱包）   │
│  NetworkSwitcher, WalletOpsPanel                │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│           Hooks 层 (React Hooks)                  │
│  useWallet, useSIWE                              │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│         Store 层 (Zustand Store)                  │
│  authStore (认证 + 钱包状态)                      │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│        适配器层 (wallet-adapter)                  │
│  getWalletClient, requestPrimaryAccount         │
│  signMessage, sendTransaction                   │
└──────────────────────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────┐
│          SDK 层 (@trustaid/wallet-sdk)            │
│  createWalletClient, EmbeddedKeystore           │
└──────────────────────────────────────────────────┘
```

### 1.3 支持的钱包类型

#### Injected 模式（注入式钱包）

**代表钱包**: MetaMask、Rabby、Coinbase Wallet

**特点**:
- ✅ 通过浏览器插件注入 `window.ethereum`
- ✅ 账户管理由插件负责
- ✅ 签名时弹出插件窗口
- ✅ 支持网络切换

**使用场景**:
```typescript
// 用户点击 MetaMask
setWalletMode("injected");
await signIn(); // 弹出 MetaMask 请求授权
```

#### Embedded 模式（内置钱包）

**代表**: TrustAid 本地钱包

**特点**:
- ✅ 私钥本地加密存储（AES-GCM）
- ✅ 密码解锁访问
- ✅ 支持 HD 钱包派生
- ✅ 可导入/导出助记词

**使用场景**:
```typescript
// 用户创建本地钱包
setWalletMode("embedded");
setEmbeddedWalletConfig({ password });
await createOrImportEmbeddedWallet({ password });
```

## 二、核心 Store 与状态管理

### 2.1 authStore 详解

**源码位置**: `frontend/store/authStore.ts`

#### 2.1.1 状态字段

```typescript
type AuthState = {
  // ---------- 认证相关 ----------
  token: string | null;          // JWT 访问令牌
  address: string | null;        // 用户钱包地址
  expiresAt: number | null;      // token 过期时间戳
  isLoading: boolean;            // 加载状态

  // ---------- 钱包相关 ----------
  walletRuntime: "injected" | "embedded" | null; // 钱包运行环境
  chainIdHex: string | null;     // 链 ID（十六进制）
  walletChainId: number | null;  // 链 ID（十进制）
  walletSessionState:            // 会话状态机
    | "idle"
    | "locked"
    | "unlocked"
    | "pendingApproval"
    | "connecting"
    | "signing"
    | "submitted"
    | "failed";
  isConnecting: boolean;         // 是否正在连接
  error: string | null;          // 错误信息

  // ---------- 角色相关 ----------
  roles: RoleId[];               // 用户角色列表
  activeRole: RoleId | null;     // 当前激活角色
};
```

#### 2.1.2 状态机流转

```
idle (初始状态)
  ↓
用户点击连接
  ↓
connecting (连接中)
  ↓
请求账户成功
  ↓
unlocked (已解锁/已连接) ←──┐
  ↓                        │
用户断开/异常              │重连
  ↓                        │
locked (已锁定) ───────────
```

#### 2.1.3 Actions 详解

##### `setSession` - 设置登录会话

```typescript
setSession: (token, address, expiresAt) =>
  set({ token, address, expiresAt, isLoading: false })
```

**使用示例**:
```typescript
// SIWE 登录成功后
const expiresAt = Date.now() + 3600000; // 1 小时后过期
setSession(token, address, expiresAt);
```

##### `clearSession` - 清除会话（退出登录）

```typescript
clearSession: () =>
  set({
    token: null,
    address: null,
    expiresAt: null,
    isLoading: false,
    walletRuntime: null,
    chainIdHex: null,
    walletSessionState: "idle",
    roles: [],
    activeRole: null,
  })
```

**使用示例**:
```typescript
// 用户点击退出
<Button onClick={clearSession}>退出</Button>
```

##### `updateWallet` - 批量更新钱包状态

```typescript
updateWallet: (updates) => set(updates)
```

**使用示例**:
```typescript
// 连接成功后批量更新
updateWallet({
  walletRuntime: "injected",
  walletChainId: 1,
  walletSessionState: "unlocked",
  isConnecting: false,
  error: null
});
```

#### 2.1.4 派生状态（计算属性）

##### `isAuthenticated` - 是否已认证

```typescript
isAuthenticated: () => {
  const { token, expiresAt } = get();
  if (!token || !expiresAt) return false;
  return Date.now() < expiresAt;
}
```

**使用示例**:
```typescript
const isAuthenticated = useAuthStore(state => state.isAuthenticated());

if (!isAuthenticated) {
  // 跳转到登录页
}
```

##### `isWalletConnected` - 钱包是否已连接

```typescript
isWalletConnected: () => {
  const { address, walletSessionState } = get();
  return !!address && walletSessionState === 'unlocked';
}
```

**使用示例**:
```typescript
const isWalletConnected = useAuthStore(state => state.isWalletConnected());

if (!isWalletConnected) {
  // 显示连接钱包按钮
}
```

### 2.2 持久化策略

#### 2.2.1 持久化配置

```typescript
export const useAuthStore = create<AuthState>()(
  persist(
    (set, get) => ({
      // ... store definition
    }),
    {
      name: 'trustaid-auth-storage', // LocalStorage 键名
      partialize: (state) => ({
        // 仅持久化非敏感信息
        address: state.address,
        roles: state.roles,
        activeRole: state.activeRole,
        walletRuntime: state.walletRuntime,
        walletChainId: state.walletChainId,
      }),
      storage: createJSONStorage(() => localStorage),
    }
  )
);
```

#### 2.2.2 安全考虑

**持久化字段**:
- ✅ `address` - 钱包地址（公开信息）
- ✅ `roles` - 角色列表（公开信息）
- ✅ `walletRuntime` - 钱包类型（公开信息）

**不持久化字段**:
- ❌ `token` - JWT 令牌（敏感，过期需重新登录）
- ❌ `walletSessionState` - 会话状态（每次重新建立）
- ❌ `error` - 错误信息（临时状态）

## 三、钱包 Hooks 详解

### 3.1 useWallet Hook

**源码位置**: `frontend/hooks/useWallet.ts`

**功能**: 管理钱包连接状态、提供连接/断开方法

#### 3.1.1 返回值

```typescript
type WalletReturn = {
  // 状态
  address: string | null;        // 钱包地址
  isConnected: boolean;          // 是否已连接
  isConnecting: boolean;         // 是否正在连接
  error: string | null;          // 错误信息
  errorCode: string | null;      // 错误码
  chainId: number | null;        // 链 ID
  runtime: "injected" | "embedded" | null; // 运行环境
  
  // 操作
  connect: () => Promise<void>;      // 连接钱包
  disconnect: () => Promise<void>;   // 断开连接
  clearError: () => void;            // 清除错误
  retry: () => Promise<void>;        // 重试连接
  formatAddress: () => string;       // 格式化地址
  switchNetwork: (chainId: number) => Promise<void>; // 切换网络
};
```

#### 3.1.2 连接逻辑

```typescript
const connect = useCallback(async () => {
  // 1. 防止重复连接
  if (isConnectingRef.current) return;
  
  isConnectingRef.current = true;
  setIsConnecting(true);
  clearError();
  setWalletSessionState('connecting');

  // 2. 设置连接超时（30 秒）
  connectTimeoutRef.current = setTimeout(() => {
    setError('连接超时，请检查网络或钱包插件');
    setWalletSessionState('failed');
  }, CONNECT_TIMEOUT_MS);

  try {
    // 3. 请求主账户
    const addr = await requestPrimaryAccount();
    
    // 4. 连接成功，清除定时器
    clearTimers();
    console.log('[useWallet] 连接成功:', formatAddressUtil(addr));
  } catch (err) {
    // 5. 错误处理
    const mappedError = mapWalletError(err);
    setError(mappedError.message);
    throw mappedError;
  } finally {
    setIsConnecting(false);
    isConnectingRef.current = false;
  }
}, []);
```

**使用示例**:
```typescript
const { connect, isConnected, isConnecting } = useWallet();

<Button
  onClick={connect}
  disabled={isConnected || isConnecting}
>
  {isConnecting ? '连接中...' : isConnected ? '已连接' : '连接钱包'}
</Button>
```

#### 3.1.3 事件订阅

```typescript
useEffect(() => {
  const client = getWalletClient();
  if (!client) return;
  
  // 订阅钱包事件
  unsubscribe = client.subscribe({
    onAccountsChanged: (accounts) => {
      const primary = String(accounts[0] || '').toLowerCase();
      if (primary) {
        setAddress(primary); // 更新地址
        clearTimers(); // 停止重连
      } else {
        handleDisconnect(); // 触发重连
      }
    },
    onChainChanged: (chainIdHex) => {
      setWalletChainId(chainIdHex); // 更新链 ID
    },
    onDisconnect: () => {
      handleDisconnect(); // 触发重连
    },
  });
  
  return () => unsubscribe();
}, []);
```

**事件处理策略**:

| 事件 | 触发条件 | 处理逻辑 |
|------|----------|----------|
| `onAccountsChanged` | 用户切换账户 | 更新地址、停止重连 |
| `onAccountsChanged` (空) | 用户断开连接 | 触发自动重连 |
| `onChainChanged` | 用户切换网络 | 更新链 ID |
| `onDisconnect` | 钱包断开 | 触发自动重连 |

#### 3.1.4 自动重连机制

```typescript
const handleDisconnect = useCallback(() => {
  setWalletSessionState('locked');
  setAddress(null);
  setWalletChainId(null);
  clearTimers();
  
  // 如果不是用户主动断开，启动自动重连
  if (walletSessionState !== 'locked' && retryCount < MAX_RETRY_COUNT) {
    reconnectTimerRef.current = setTimeout(() => {
      setRetryCount(prev => prev + 1);
      connect();
    }, RECONNECT_INTERVAL_MS);
  }
}, []);
```

**重连参数**:
- `RECONNECT_INTERVAL_MS = 5000` (5 秒间隔)
- `MAX_RETRY_COUNT = 3` (最多 3 次)
- `CONNECT_TIMEOUT_MS = 30000` (30 秒超时)

### 3.2 useSIWE Hook

**源码位置**: `frontend/hooks/useSIWE.ts`

**功能**: 处理 SIWE (Sign-In with Ethereum) 登录/登出

#### 3.2.1 返回值

```typescript
type SIWEReturn = {
  address: string | null;
  busy: boolean;
  token: string | null;
  signIn: () => Promise<void>;
  signOut: () => Promise<void>;
  /** 调用后端 POST /v1/auth/refresh-roles，按 ROLES_SOURCE 重算 roles 并更新 JWT（链上 grant/切链后使用） */
  refreshRoles: () => Promise<unknown>;
  refreshBusy: boolean;
};
```

钱包页（`/wallet`）在 **切链** 后会尝试 `refreshRoles`；也可手动点击「同步链上角色」。详见仓库根目录 `docs/链上角色与权限.md`。

#### 3.2.2 登录流程

```typescript
const signIn = useCallback(async () => {
  setBusy(true);
  setError(null);
  
  try {
    // 1. 请求钱包连接
    const addr = await requestPrimaryAccount();
    
    // 2. 生成 SIWE 消息
    const message = generateSIWEMessage(addr);
    
    // 3. 请求签名
    const signature = await signMessage(message);
    
    // 4. 提交后端验证
    const token = await verifySIWE(addr, signature);
    
    // 5. 设置会话
    const expiresAt = Date.now() + 3600000;
    setSession(token, addr, expiresAt);
  } catch (error) {
    setError(mapError(error));
    throw error;
  } finally {
    setBusy(false);
  }
}, []);
```

**使用示例**:
```typescript
const { signIn, busy } = useSIWE();

<Button onClick={signIn} disabled={busy}>
  {busy ? '登录中...' : '使用钱包登录'}
</Button>
```

#### 4.1.3 布局、动效与无障碍（实现要点）

- **Portal**：`createPortal(..., document.body)`；仅在客户端渲染；`isOpen` 为 true 时挂载带 `AnimatePresence` 的根节点，便于执行 **exit** 淡出（避免原先 `!isOpen` 直接 `return null` 导致 exit 无法播放）。
- **结构**：白卡片 **flex 列**；**顶栏**为 **可选「← 返回」**（已进入 MetaMask / 内置 / WalletConnect 子流程时显示，`selectWallet(null)` 回到连接方式选择）+ 标题（`aria-labelledby`）+ 关闭钮；内容区可滚动（`max-h` + `overflow-y-auto`）。内置钱包在 **SIWE 阶段**另有一条内容区 **「上一步」**，回到密码表单（`setEmbeddedStage({ stage: 'form' })`），与顶栏「返回上一级」语义区分。
- **样式**：选项卡与输入等统一使用 Tailwind 扩展的 **`rounded-card` / `rounded-input`**，图标衬底 **`bg-surface`**；主按钮不在外层再叠 `disabled:opacity-*`，避免与 `Button` 主色禁用态双重弱化；内置「进入钱包」在密码为空时展示 **「请先输入密码」**，并以 **`aria-describedby`** 关联主按钮。
- **定位**：遮罩 `fixed inset-0 z-[100]`；关闭钮在卡片顶栏内，不再依赖未设 `relative` 的错位 `absolute`。
- **滚动与焦点**：打开时 `document.body.style.overflow = 'hidden'`；打开后将焦点移至关闭按钮；关闭时在 effect 清理中 **restore** 至打开前的 `document.activeElement`。
- **减弱动效**：Framer `useReducedMotion()` 为真时缩短 `duration`、弱化位移。
- **步骤切换**：内层 `AnimatePresence mode="wait"` + `key={selectedWallet ?? 'picker'}`，避免步骤 DOM 复用导致动画异常。

#### 4.1.4 支持的钱包选项

含 MetaMask、本地内置钱包、WalletConnect（需 `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID`）。图标见 `features/wallet/Svg.tsx`（WalletConnect 为线稿主色风格，与品牌一致）。

#### 4.1.5 Embedded 钱包状态机

```typescript
type EmbeddedPhase =
  | { stage: "form" }
  | { stage: "mnemonic_backup"; address: string; mnemonic: string } // 仅「新建」：展示助记词一次
  | { stage: "ready"; address: string }
  | { stage: "siwe_error"; address: string; msg: string };
```

**状态流转**:

```
form（解锁 / 新建 / 导入）
    ↓
新建：链上创建成功 → mnemonic_backup（用户确认已备份）→ ready
解锁 / 导入：直接进入 ready
    ↓
SIWE 签名
    ↓
成功 → 关闭模态框；失败 → siwe_error
```

#### 4.1.6 密码强度检测

```typescript
useEffect(() => {
  if (!password) { setPasswordStrength(0); return; }
  let s = 0;
  if (password.length >= 8) s++;      // 长度 >= 8
  if (password.length >= 12) s++;     // 长度 >= 12
  if (/[A-Z]/.test(password)) s++;    // 包含大写字母
  if (/\d/.test(password)) s++;       // 包含数字
  if (/[^A-Za-z0-9]/.test(password)) s++; // 包含特殊字符
  setPasswordStrength(s);
}, [password]);
```

**强度等级**:

| 分数 | 等级 | 颜色 | 说明 |
|------|------|------|------|
| 0-1 | 弱 | 红色 | 密码过于简单 |
| 2-3 | 中 | 黄色 | 密码强度一般 |
| 4-5 | 强 | 绿色 | 密码强度高 |

### 4.2 连接入口与 WalletConnect（已统一 wallet-sdk）

原独立组件 `WalletSelector`（wagmi connectors）已移除。所有连接均通过 **`ConnectWalletModal`** + **`useConnectWallet`**：

- **MetaMask**：`resetExternalWalletProvider` 后 `setWalletMode('injected')`，`authStore.walletConnector = 'injected'`，再走 SIWE。
- **WalletConnect**：`@walletconnect/ethereum-provider` 初始化后，经 `setWalletProviderResolver` 注入 `createWalletClient({ providerResolver })`，`walletConnector = 'walletconnect'`，再走 SIWE。需配置环境变量 **`NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID`**（WalletConnect Cloud）。
- **内置钱包**：`setWalletMode('embedded')`，密码与 RPC 仅用于本地 keystore / JsonRpc，**切链**在 `wallet-adapter.switchNetwork` 内仅更新 `embeddedConfig` 与 store，不调用 `wallet_switchEthereumChain`。

共享链列表见 `frontend/lib/wallet/networks.ts`（与 `NetworkSwitcher` 一致）。

const [bindForm, setBindForm] = useState({ 
  mainAddr: "", 
  newAddr: "", 
  proof: "" 
});

async function onBind(event: FormEvent) {
  event.preventDefault();
  if (!siweAddress) {
    setFormError("请先连接钱包（SIWE）再进行钱包绑定");
    return;
  }
  try {
    const main = bindForm.mainAddr || siweAddress;
    requireEthAddress(main, "mainAddr");
    requireEthAddress(bindForm.newAddr, "newAddr");
    requireNonEmpty(bindForm.proof, "proof");
    const data = await bindWallet({ ...bindForm, mainAddr: main });
    setResult(JSON.stringify(data, null, 2));
  } catch (error) {
    setFormError(toUserErrorMessage(error));
  }
}

return (

    <h2 className="mb-3 section-title">钱包绑定</h2>
    <form onSubmit={onBind} className="space-y-3">
      <Input label="主钱包地址" value={bindForm.mainAddr} onChange={...} />
      <Input label="新钱包地址" value={bindForm.newAddr} onChange={...} />
      <Input label="绑定证明" value={bindForm.proof} onChange={...} />
